Reusable HTML Components

1. Why make my own reusable HTML components ?

During my training, I had to create a portfolio website without any framework. I ended up with a lot of code repetions inside my html and wished I could have reusable elements. Thus the following exercice!

I'm not trying to reinvent the wheel, I just think it's a nice learning challenge.

2. Theory

1. Simple Components

Simple components have a fixed content. They can take attributes such as id or class and can be styled with external css.

Their attibute values can be tracked (have a look at the console).

If a value is given inside the component definition, that value overrides given values.

Components can be interacted with.
